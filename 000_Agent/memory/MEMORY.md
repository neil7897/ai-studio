## 用戶偏好

### 工作與溝通風格
- 先看結論再看細節（BLUF）
- 假日／晚上也會工作，非典型朝九晚五
- 偏好條列取代長段落
- 不喜歡來回多次確認，問清楚一次就執行

### 常用工具與平台
- 客戶溝通：Line / WhatsApp
- 社群主戰場：Facebook、Instagram
- 影音產出：YouTube、Podcast
- 社群排程工具：Meta Business Suite（免費、官方、FB + IG 一起管）

### 公司與業務背景
- 公司：福昌紹紀企業有限公司（一人公司 / 負責人）
- 客戶類型：B2B 企業客戶為主
- 行銷模式：圖文內容行銷導流

### 內容創作偏好
- 口語化、有親近感，不要太正式
- 有真實故事或案例帶入
- 簡短有力，不廢話
- 開頭用問句或 Hook 吸引人讀下去

## 業務常見情境

### 社群內容排程
- 平台：Facebook + Instagram
- 頻率：每週 3–5 次
- 常見需求：一次批量產出多篇貼文（5–10 篇），供排程使用
- AI 操作原則：產出時附上建議發文日期或主題分類，方便直接排進行事曆

### 快速大量文字產出
- 觸發時機：社群貼文批次、客戶 Email / Line 訊息、活動文案
- AI 操作原則：
  - 一次產出多篇時，先列標題清單讓陛下確認方向再展開全文
  - 預設符合內容創作偏好（口語、有故事、簡短有力、Hook 開頭）
  - 長稿同步存至 `100_Todo/drafts/` 對應子資料夾

## Feedback（AI 學到的原則）

### 禁止行為
| 錯誤做法 | 正確做法 | 原因 |
| :--- | :--- | :--- |
| 在回答結尾做總結重述 | 直接結束，不重複 | 陛下看得到內容，不需要 AI 再唸一遍 |
| 使用表情符號 | 純文字輸出 | 除非陛下明確要求 |
| 每個步驟都停下來問確認 | 問清楚一次，然後執行到底 | 頻繁確認打斷節奏，浪費時間 |

### 保留行為
- **先問清楚再動手**：需求模糊時用 AskUserQuestion 釐清，不要靠猜硬做
- **推薦優先、選項其次**：有多方案時先推薦一個並說理由，再列其他供選擇
- **直接給可執行版本**：文案、程式碼、計畫直接給成品，不給概念或大綱
- **主動存檔**：長稿、草稿自動存進對應資料夾，不用陛下另外交代
